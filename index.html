PROJECTS

JaxComedy

During my work as a food server at the Comedy Club of Jacksonville, I approached the owner about creating an Android application for the business. My goal with the application was to provide an easy and informative way for users to access information and to have access to a rewards program to promote business. I designed the application to synchronize data with a private Google Spreadsheet which contained information about each show including the performer and showtimes. This information was used to send weekly notifcations to the user, which were incentivized by granting reward points upon viewing. Points were also awarded by presenting the device to the host or hostess who entered a password to confirm ticket purchases; these points would accumulate and could be exchanged for free tickets to shows. The app was praised by the owner and implemented for a short while before the Comedy Club closed temporarily.

The application itself was written in Java and XML and uses JSON parsing, networking via Google Sheets synchronization, a custom version of the Cards user interface as a part of Google's Material Design, and other Android-specific knowledge including activities, intents, adapters, and the Android life cycle.

The application is available to download (though no longer maintained) from Google Play, and the source code is available on github.

BattleShip

I was a part of a group project team that decided to recreate the board game BattleShip in command line using C++. The game uses object-orientation, multi-dimensional arrays, pointers, stacks, file reading and string parsing, and other general C++ knowledge. The program offers a guided tutorial, and can be played against either another player or an AI opponent with three difficulties.

My contribution included creating different ship placement functions (place ships at specified coordinates, place them in predefined locations for bug testing, or place them randomly on the board), validation for ship placements, sunken ship confirmation, a guided tutorial, and parts of class definitions and opponent AI, which utilized stacks to simulate strategic play.

The game's source code is available to download through github (it may need to be run with 'g++ -std=c++0x' on UNIX systems).

MyGymBuddy

MyGymBuddy was an attempt to practice with Google's Material Design guidelines and release a product to Google Play with the intent to monetize. It was originally meant to be a fully customizable workout tracker for cardio and weightlifting, but I stopped development to focus on school and a family business and don't intend on continuing development soon.

The application, similar to the JaxComedy Android app, utilizes Java, XML, and similar Android-specific knowledge. I followed elements of Google's Material Design, used XML layouts to build custom Dialogs, and used the Gson library to handle saving and loading class data to the user's device.

The application's source code is available on github.
